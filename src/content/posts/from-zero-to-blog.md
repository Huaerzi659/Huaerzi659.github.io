今天，我终于有了一个真正可以被别人打开的个人博客。

它不只是一个网页，也不只是一个仓库。对我来说，它更像是一个被正式放在互联网上的小房间：门牌写着 **Waltz, Then Think**，里面暂时还不算热闹，但已经有灯，有桌子，也有以后慢慢摆上去的东西。

最开始我其实没有多清楚自己要做什么。我只知道，我想要一个属于自己的地方。它不用一开始就很完整，也不用马上变成作品集、技术博客、简历主页或者什么很标准的东西。它可以先存在，先被打开，先成为一个能承接想法的容器。

于是第一版就从很朴素的目标开始：免费上线、别人能访问、以后能用 Markdown 写文章。

这个选择让我喜欢的地方在于，它没有把事情变得太重。博客用 Astro 搭起来，文章放在 `src/content/posts/` 里，每一篇都是一个 Markdown 文件。写完之后推到 GitHub，GitHub Actions 会自动构建，最后由 GitHub Pages 发布出去。

这套流程听起来像一串技术名词，但真正理解之后，它其实很温柔：

- Markdown 负责让我专心写字。
- Astro 负责把文字变成页面。
- GitHub 负责保存代码和文章。
- GitHub Pages 负责把它公开给世界。

我喜欢这种感觉。一个想法从脑子里出来，先变成文字，再变成文件，然后变成一个可以通过链接抵达的页面。它不再只是“我以后想做点什么”，而是已经变成了“我做了一点什么”。

当然，第一版不会完美。现在还没有评论系统，也没有复杂的后台，更没有很花哨的功能。但这反而让我觉得安心。因为我不想一开始就被功能牵着走。比起先搭一个巨大的架子，我更想先把这里用起来。

以后这里会写技术，也会写经验和感想。可能有一些文章很具体，记录某个 bug、某个项目、某个工具；也可能有一些文章更松散，只是某天突然想明白的一件小事。

我希望这个博客慢慢长成一个索引：记录我解决问题的方式，也记录我理解自己的方式。

先跳一步，再想一想。

这大概就是它开始的原因。
