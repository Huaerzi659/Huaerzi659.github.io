# UI 降噪与阅读层级优化设计方案

## 背景

当前博客已经形成暗色霓虹赛博朋克方向，也有主题索引、RSS、动效和回到顶部控件。但整体视觉仍有一点“别扭”：许多模块同时使用发光、边框和 HUD 元素，导致页面个性足够强，但视觉重心不够清晰，阅读区域不够耐看。

## 目标

- 保留赛博朋克暗色霓虹风格，不改回浅色或杂志模板风。
- 降低无差别发光，让霓虹只服务于关键状态、主标题和交互反馈。
- 提升首页 hero 的呼吸感和主次层级。
- 让文章卡片更像博客入口：更易扫读、更少控制台感。
- 保持无新依赖，继续使用 Astro 组件 CSS 和全局 CSS。
- 继续尊重 `prefers-reduced-motion`，不新增重滚动特效。

## 主要问题判断

1. 霓虹层级太平均：hero、面板、卡片、标签、页脚都在发光，导致主视觉和内容列表互相抢注意力。
2. 文章卡片阅读感偏弱：卡片 hover 和边框很醒目，但标题、摘要、标签之间的信息节奏还不够像“文章入口”。
3. 首页 hero 左右关系偏硬：左侧大标题和右侧 desk-panel 都很强，两者没有形成“主角 + 辅助仪表”的自然关系。
4. 黄色 thinking note 和绿色 live 状态增加了额外色相，容易让主色从 cyan/pink 发散。

## 方案

采用“霓虹降噪 + 阅读层级重排”。

### 全局视觉

- 保持深色背景、网格和扫描线，但降低全局装饰透明度。
- 将 cyan/pink 作为主要霓虹色，减少绿色、黄色在大面积组件中的存在感。
- 新增更克制的 surface token，用于普通内容卡片；强 glow 只用于 hover/focus 或少量主视觉。
- 统一按钮、卡片、标签的 transition 使用现有 motion token。

### 首页 Hero

- 左侧 identity-panel 继续作为第一视觉，但降低 HUD 图层和 scanline 亮度。
- 右侧 desk-panel 从“同等强度面板”降为辅助信息模块，减少阴影和强边框。
- 调整 hero 高度和间距，让标题、说明、按钮、状态条之间更有呼吸感。
- thinking note 从黄色警示感改成低调的 cyan/pink 暗面板。

### 文章卡片

- 默认状态更安静：降低背景渐变、边框和阴影。
- 标题权重保持清晰，摘要颜色略提亮，提升可读性。
- meta 信息和标签退到辅助层，减少抢眼程度。
- hover/focus 时再显示霓虹边框和扫光，作为明确交互反馈。

### Section 节奏

- 首页“最近写下”“精选思考”“主题索引”之间保持清晰分段，但减少每段都像独立强面板的感觉。
- 文章列表页继续使用同一 ArticleCard，视觉收益同步覆盖。
- 主题面板保留入口感，但降低背景发光，让 tags 更像可扫读索引。

## 不做什么

- 不加更多大面积动画、滚动劫持、视差或页面转场。
- 不改站点定位和文案人格。
- 不新增图片、字体或前端依赖。
- 不重构内容模型和路由。

## 验证

- 扩展 UI/UX 静态验证，检查降噪 token、hero 降噪类、ArticleCard 静态层级和 reduced-motion 仍存在。
- 本地运行 `verify:ux`、`verify:motion`、`verify:rss`。
- 本地运行 Astro build。
- 检查构建产物中首页和文章页仍包含回到顶部控件、文章卡片和主题入口。
