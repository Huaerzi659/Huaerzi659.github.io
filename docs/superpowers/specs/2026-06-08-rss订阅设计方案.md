# RSS 订阅设计方案

## 背景

`Waltz, Then Think` 已经有稳定的 Astro + Markdown 内容结构，文章集中在 `src/content/posts/`，并通过内容集合读取。新增 RSS 的目标是让读者可以用 Feedly、Inoreader、NetNewsWire 等阅读器订阅博客更新。

## 方案

采用 Astro 官方 `@astrojs/rss` 包生成标准 RSS feed。新增 `src/pages/rss.xml.js`，让站点在 `https://Huaerzi659.github.io/rss.xml` 输出 RSS。

每条 RSS item 来自 `posts` 内容集合，并包含：

- `title`: 文章标题
- `description`: 文章摘要
- `pubDate`: 文章日期
- `link`: 文章公开路径 `/articles/<slug>/`

文章按发布日期倒序排列，和站内文章列表保持一致。

## 自动发现

在全站基础布局的 `<head>` 中添加：

```html
<link rel="alternate" type="application/rss+xml" title="Waltz, Then Think RSS" href="https://Huaerzi659.github.io/rss.xml" />
```

这样 RSS 阅读器可以从站点首页自动识别订阅地址。

## 文档

在 `README.md` 中补充 RSS 地址，方便后续维护和公开说明。

## 验证

新增一个本地验证脚本，检查构建后的 `dist/rss.xml`：

- 文件存在
- XML 中包含站点标题
- XML 中包含每篇文章链接
- XML 中包含近期文章标题
- XML item 数量与 Markdown 文章数量一致

最后运行 `npm run build` 和 RSS 验证脚本。
