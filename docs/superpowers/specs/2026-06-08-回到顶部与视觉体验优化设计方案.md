# 回到顶部与视觉体验优化设计方案

## 背景

当前博客已经完成主题索引、RSS、Pages workflow 清理和轻量赛博朋克动效增强。下一步重点是提升日常浏览体验：让读者在文章页和较长列表页里更容易回到顶部，同时继续微调暗色霓虹界面的层次和交互反馈。

## 目标

- 添加全站可用的“回到顶部”控件。
- 保持控件小巧、清晰，不遮挡正文阅读。
- 延续赛博朋克暗色霓虹风格。
- 继续尊重 `prefers-reduced-motion`。
- 不引入新依赖，不把页面改成重动效或滚动劫持体验。
- 优化已有链接、按钮、页脚和面板的交互一致性。

## 方案

采用“实用增强 + 轻视觉优化”方案。

新增一个 `BackToTop.astro` 组件，并挂载到 `BaseLayout.astro`，让首页、文章页、主题页、关于页都能使用。按钮固定在右下角，滚动超过一段距离后显示；点击后回到页面顶部。控件使用霓虹边框、半透明暗色背景和向上箭头图标，保持 HUD 小按钮的感觉。

视觉优化聚焦共用层级：

- 全局开启 `scroll-behavior: smooth`，并在 reduced motion 下关闭。
- 为浮动控件增加稳定尺寸、可见 focus 状态和移动端安全间距。
- 微调页脚链接、通用按钮、文章卡片和标签的 transition token，使交互节奏更统一。
- 增加轻量页面底部 glow/edge 细节，让长页面结束位置更完整。

## 交互细节

- 默认隐藏“回到顶部”按钮。
- 页面滚动超过约 360px 后显示按钮。
- 点击按钮时，如果用户没有开启 reduced motion，则平滑滚动到顶部；如果开启 reduced motion，则直接跳转。
- 按钮使用真实 `<button>`，包含 `aria-label="回到页面顶部"`。
- JavaScript 只负责监听滚动、切换显示状态和触发滚动，不承担内容渲染。

## 可访问性

- 控件可键盘聚焦，focus 样式沿用全站霓虹轮廓。
- 按钮面积不小于 44px，适合触屏点击。
- 不依赖颜色表达唯一状态。
- 视觉动效保持短促，并受 `prefers-reduced-motion` 约束。

## 验证

- 新增或扩展静态验证脚本，检查 `BackToTop.astro`、layout 挂载、ARIA 文案、reduced motion 处理和关键 CSS 类。
- 本地运行 Astro build。
- 本地运行 RSS 验证，确保 RSS 不受 UI 改动影响。
- 本地运行 motion/UX 验证。
- 可用时检查公网首页和文章页包含回到顶部控件。
