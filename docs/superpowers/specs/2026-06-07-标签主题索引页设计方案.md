# Waltz, Then Think 标签页 / 主题索引页设计方案

日期：2026-06-07

## 背景

当前博客已经具备首页、文章列表、文章详情、关于页，以及 Markdown 内容集合。文章 frontmatter 中已经有 `tags` 字段，`src/lib/posts.ts` 也已经提供 `getAllTags(posts)`，首页和文章列表页已经用 `TagPill` 展示主题入口的雏形。

这次要补齐的是独立的标签页 / 主题索引页，让访客可以从“我想看某类内容”出发浏览博客，而不是只能按时间线或文章列表进入。

站点视觉方向保持现有的赛博朋克暗色霓虹，不回到杂志风或浅色模板风。

## 目标

- 新增一个完整的 `/topics/` 主题索引页。
- 新增每个主题的详情页 `/topics/[slug]/`。
- 让首页、文章列表页、文章详情页中的标签成为可以继续探索的站内入口。
- 复用现有 Astro Content Collections、Markdown、`ArticleCard`、`TagPill` 和全局霓虹视觉 token。
- 保持页面可静态生成，适配 GitHub Pages。
- 不公开强调博客主人的学生身份。

## 非目标

- 不做客户端全文搜索。
- 不做动态筛选、状态管理或 JavaScript 应用化。
- 不引入新依赖。
- 不做标签重命名后台。
- 不做复杂动画、闪烁故障特效或影响阅读的装饰。
- 不把文章标签替换成复杂分类系统；标签仍保持轻量、灵活。

## UI/UX Pro Max 参考结果

已按项目约定读取 `.codex/skills/ui-ux-pro-max/`，并用本地数据查询：

- `personal developer blog dark neon cyberpunk topic index readable`
- `topic index tag archive navigation dark`
- `dark neon readable accessibility animation`
- `layout responsive links cards focus`

采纳结果：

- 深色高对比背景适合开发者个人博客。
- 可采用块状信息面板、霓虹边框、运行绿和青/洋红强调色。
- 可点击卡片和链接必须有 hover 与 focus-visible 状态。
- 移动端正文和列表文字不小于 16px。
- 尊重 `prefers-reduced-motion`，不做无限装饰动画。

不采纳结果：

- 不采用 webinar registration 结构。
- 不采用手写字体。
- 不采用大面积营销式 CTA。
- 不为了“活泼”牺牲博客阅读和索引效率。

## 推荐方案

采用完整主题索引方案：

1. `/topics/` 展示所有主题、每个主题文章数量、最近更新时间，以及每个主题下的最新文章预览。
2. `/topics/[slug]/` 展示某一个主题下的所有文章。
3. Header 新增“主题”导航。
4. 首页和文章列表页里的主题 chip 链接到对应主题详情页。
5. 文章详情页的标签也链接到对应主题详情页。
6. `ArticleCard` 内部仍然只展示非链接标签，避免在整张文章卡链接里嵌套链接。

备选方案是只做 `/topics/` 单页，不做单主题详情页。该方案实现更少，但文章数量增长后定位能力不足，因此不采用。

## 信息架构

### `/topics/`

页面分三层：

- 顶部 hero：说明这是主题索引，显示文章数与主题数。
- 主题矩阵：每个主题一张轻量索引卡，包含主题名、文章数、最新更新时间和简短文章标题预览。
- 分组列表：按主题列出文章卡，便于从一个主题继续阅读。

页面文案应保持个人博客语气，例如“这些入口不是严格分类，更像思考留下的坐标”。

### `/topics/[slug]/`

页面分三层：

- 顶部 hero：显示当前主题名、文章数量、最近更新时间。
- 文章列表：使用现有 `ArticleCard` 展示该主题下的所有文章，按日期倒序。
- 返回入口：提供回到 `/topics/` 的链接。

不存在的主题 slug 不生成静态路径，因此 GitHub Pages 只发布真实主题页。

## 数据设计

在 `src/lib/posts.ts` 增加轻量工具函数：

- `slugifyTag(tag: string)`: 将标签转成 URL 安全 slug。
- `getTagHref(tag: string)`: 返回 `/topics/<slug>/`。
- `getPostsByTag(posts, tag)`: 按标签筛选文章并按日期倒序。
- `getTagSummaries(posts)`: 返回主题汇总，包含 `tag`、`slug`、`href`、`count`、`latestDate`、`posts`。
- `findTagBySlug(posts, slug)`: 用于动态主题页从 slug 找回原始标签名。

当前标签为中文，`slugifyTag` 保留 `tag.trim()` 的原始文本，供 Astro `getStaticPaths()` 匹配动态路由；`getTagHref` 再用 `encodeURIComponent` 输出浏览器友好的链接。这样不需要手写映射表，也能兼容未来英文标签。

## 组件设计

### `TagPill`

保留默认静态 chip，同时新增可选属性：

- `href?: string`
- `count?: number`

如果传入 `href`，渲染为 `<a>`；否则渲染为 `<span>`。视觉保持一致，链接态增加 `cursor: pointer`、hover 边框和 focus-visible 支持。

### `ArticleCard`

不改变结构和 props。因为整张卡片本身是链接，内部标签保持静态 `TagPill`，避免嵌套交互元素。

### 新页面局部结构

不新增复杂共享组件。主题页的索引卡和列表样式先放在对应 `.astro` 页面内，保持范围清晰。等未来主题页扩展到搜索、推荐或归档统计，再抽组件。

## 页面与导航

修改 Header：

- 导航从“文章 / 关于 / GitHub”变为“文章 / 主题 / 关于 / GitHub”。
- 移动端由三列改为四列，避免文字挤压和换行跳动。

修改首页与文章列表页：

- 主题索引区域中的 `TagPill` 链接到 `/topics/[slug]/`。
- 与现有霓虹 chip 风格保持一致。

修改文章详情页：

- 文章 header 中的标签链接到对应主题页。
- 这不会产生嵌套链接，因为文章详情页的标签不在外层 `<a>` 内。

## 可访问性与响应式

- 所有主题链接可键盘聚焦。
- 主题卡片使用真实链接，不只依赖颜色表达可点击。
- hover 不改变布局尺寸。
- 主题矩阵在桌面端使用 2 到 3 列，移动端自然变成单列。
- 375px 宽度无横向滚动。
- 正文与卡片文字保持现有高对比暗色可读性。
- 不新增无限动画。

## 验收标准

- `npm run build` 或等价 Astro build 成功。
- `dist/topics/index.html` 生成并包含所有现有标签。
- 每个现有标签都有对应静态页面，构建产物目录为中文标签名，公开链接可使用浏览器编码后的 URL：
  - `/topics/技术/`
  - `/topics/经验/`
  - `/topics/感想/`
  - `/topics/灵感/`
- 首页主题区、文章列表页主题区、文章详情页标签都能链接到主题页。
- Header 有“主题”导航。
- 移动端 375px 不出现横向滚动。
- 视觉保持赛博朋克暗色霓虹：暗底、霓虹青/洋红、HUD 式边框、克制发光。

## 风险与处理

- 中文主题链接在 HTML 中会以编码后的 URL 输出，Astro 构建产物会生成中文目录。这是静态站最轻量可靠的方案，浏览器和 Astro 均能正常处理。
- 如果未来需要更漂亮的英文 slug，可以在内容 schema 中新增 `tagAliases` 或独立标签配置文件；本轮不引入该复杂度。
- 本地没有 `git` 和 `gh`，提交与发布阶段优先使用 GitHub connector。
